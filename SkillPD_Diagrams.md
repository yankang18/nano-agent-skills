## 5. 三级披露状态流转图

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              状态定义                                         │
└─────────────────────────────────────────────────────────────────────────────┘

   ┌─────────────┐
   │    初始态    │
   │  (Initial)  │
   └──────┬──────┘
          │
          │ 启动扫描
          ▼
   ┌─────────────┐
   │  Level 1态  │ ◄─────────────────────────────────────────────┐
   │  (Metadata) │                                               │
   │             │  特征：                                        │
   │  元数据常驻   │  - Skills Registry 在系统提示词中               │
   │  内存        │  - Token 消耗最低 (~100 tokens/skill)          │
   └──────┬──────┘  - 用于意图匹配                                 │
          │                                                      │
          │ 用户输入匹配 skill.description                         │
          │ (如：用户说"审查代码"匹配到                              │
          │  "code-reviewer: 在代码提交前进行审查...")               │
          ▼                                                      │
   ┌─────────────┐                                               │
   │  Level 2态  │                                               │
   │   (Active)  │                                               │
   │             │  特征：                                        │
   │ 完整指令加载  │  - SKILL.md 完整内容加入上下文                   │
   │ Skill 激活   │  - Token 消耗中等 (~1000+ tokens)              │
   └──────┬──────┘  - base_path 已确定                            │
          │                                                      │
          │ 按指令步骤执行                                          │
          │                                                      │
     ┌────┴────┬────────────────┐                                │
     │         │                │                                │
     ▼         ▼                ▼                                │
┌─────────┐ ┌─────────┐  ┌─────────────┐                         │
│ Level 3 │ │ Level 3 │  │   Level 3   │                         │
│ (Ref 1) │ │ (Ref 2) │  │   (Script)  │                         │
│         │ │         │  │             │                         │
│按需读取  │  │ 按需读取 │  │   按需执行    │                        │
│引用文件  │  │ 引用文件 │  │   脚本       │                         │
└────┬────┘ └────┬────┘  └──────┬──────┘                         │
     │           │              │                                │
     └───────────┴──────────────┘                                │
                 │                                               │
                 ▼                                               │
        ┌─────────────────┐                                      │
        │    Skill 完成    │                                      │
        │ (Execution Done)│                                      │
        └────────┬────────┘                                      │
                 │                                               │
                 │ 清空 current_skill_context                     │
                 │ 释放 Level 2/3 上下文                           │
                 │ (但不会从对话历史中移除)                          │
                 └───────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                          Token 消耗对比                                       │
└─────────────────────────────────────────────────────────────────────────────┘

场景：系统有 10 个 Skill，用户仅使用 code-reviewer

Level 1 (启动时):
┌────────────────────────────────────────────────────────┐
│ SkillRegistry Metadata                                 │
│ - code-reviewer: 在代码提交前进行审查... (80 tokens)       │
│ - skill-creator: 创建新技能指南...      (90 tokens)       │
│ - ... (其他 8 个技能)                   (~400 tokens)    │
│                                                        │
│ Total: ~500 tokens (常驻)                               │
└────────────────────────────────────────────────────────┘

Level 2 (触发时):
┌────────────────────────────────────────────────────────┐
│ code-reviewer/SKILL.md                                 │
│                                                        │
│ ## 工作流程                                             │
│ 1. 查阅规范：读取 `references/style-guide.md`            │
│ 2. 执行检查：运行 `bash scripts/lint.sh`                 │
│ ...                                                    │
│                                                        │
│ Total: ~1200 tokens (一次性加载)                         │
└────────────────────────────────────────────────────────┘

Level 3 (执行时):
┌────────────────────────────────────────────────────────┐
```
